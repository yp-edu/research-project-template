# Claim Guidance

- Claims are research statements the project may defend in a paper, report, or decision.
- Each claim should link to supporting or contradicting experiments, literature notes, questions, decisions, configs, results, and paper sections when useful.
- Record claim strength: tentative, supported, contradicted, or retired.
- Include limitations and missing evidence when the claim is not fully supported.
- Keep implementation details out unless they affect the validity of the claim.
- Name claim files `YYYY-MM-DD-short-title.md` using the date the claim is recorded.
- Keep category-level summaries in `README.md` factual; leave it title-only until there is something real to summarize.
- Track claim files from the category dashboard as `[claim](YYYY-MM-DD-claim-title.md): Tentative`, then replace the status as evidence accumulates.
