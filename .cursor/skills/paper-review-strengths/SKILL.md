# Paper Review: Strengths

## When to Use

- You are running a structured review pass (e.g. **review-paper** command) and need to list strengths.
- The user wants to know what is strong in the current paper or experiments.

## Instructions

- For each strength, state **what** is strong (e.g. clear problem statement, solid baselines, convincing ablations) and **where** the evidence is (section, figure, table, or code/config).
- Do not invent strengths; only list those supported by the current draft or setup.
- Output: A short list of strengths with 1–2 line justification and a pointer to the evidence (e.g. "§3.2", "Table 1", "configs/group1/first.yaml").
