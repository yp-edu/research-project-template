# Contribution Development Checklist

Before or during contribution development, ensure you:

1. Clearly state the core idea and how it differs from baselines.
2. Identify which components are new and which are reused.
3. Define hypotheses and which experiments will test them.
4. Plan ablations or controlled comparisons to isolate each design choice.
5. Ensure configs, seeds, and datasets are documented for reproducibility.
6. Anticipate reviewer questions and design experiments to address them.
