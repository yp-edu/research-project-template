# Project Memory: Schema And Guidelines

The main project memory file is **`docs/README.md`**. It should stay short and readable.

## Main Memory

`docs/README.md` should contain these sections in order:

1. **Snapshot**
2. **Research Focus**
3. **Key Artifacts**
4. **Memory Index**
5. **Next Steps**
6. **Open Questions / Decisions**

Use it for:

- Current phase
- Short project summary
- The next 1-3 actions
- High-level blockers or decisions

## Focused Memory

Use the supporting docs for durable domain memory:

- **`docs/questions/README.md`** for problem framing, research questions, hypotheses, and evaluation criteria.
- **`docs/literature/README.md`** for search focus, seed papers, themes, gaps, and notes index.
- **`docs/experiments/README.md`** for active tracks, baselines, planned runs, result index, and lessons.

## Editing Rules

- Keep each file concise.
- Update only the files relevant to the task.
- Prefer references to concrete paths over prose about where things might live.
- Do not use `docs/` as a dump for raw experiment output; keep outputs in `results/`.
